# 🧪 Plan de Pruebas de Accesibilidad – Mercado Libre

## 📄 Descripción del Proyecto

Este repositorio contiene la documentación del **Plan de Pruebas de Accesibilidad** y su **ejecución** aplicado a la plataforma de Mercado Libre.

El objetivo del proyecto fue evaluar si algunas de las funcionalidades principales del sitio web cumplen con criterios de accesibilidad que permitan su uso por personas con diferentes tipos de discapacidad.

Las pruebas se enfocaron en validar la interacción con la plataforma utilizando métodos de accesibilidad como:

- ⌨️ Navegación por teclado  
- 🔊 Compatibilidad con lectores de pantalla  
- 🏷️ Uso adecuado de etiquetas en formularios  
- 🎨 Verificación de contraste visual  

---

# 🎯 Objetivo

Ejecutar un conjunto de pruebas de accesibilidad sobre funcionalidades clave de la plataforma con el fin de validar que los usuarios puedan interactuar con el sistema de manera clara, comprensible y sin barreras tecnológicas.

---

# 📌 Alcance de las Pruebas

Las pruebas fueron realizadas sobre los siguientes módulos de la plataforma:

- 🔎 **Búsqueda de productos**
- 🛒 **Adición de productos al carrito**
- 👤 **Registro de usuarios**
- ⌨️ **Navegación mediante teclado**
- 📝 **Validación de formularios accesibles**
- 🎨 **Verificación de contraste visual**
- 🔊 **Compatibilidad con lectores de pantalla**

---

# 📚 Estándares de Accesibilidad Evaluados

Las pruebas se realizaron considerando las recomendaciones de:

- 🌐 **WCAG 2.1 (Web Content Accessibility Guidelines)**
- ⚖️ **Ley 1618 de 2013 – Colombia**
- 📜 **Decreto 1078 de 2015 – Colombia**

Estos lineamientos buscan garantizar que los servicios digitales sean accesibles para toda la población.

---

# 🛠️ Herramientas Utilizadas

Durante la ejecución del plan de pruebas se utilizaron las siguientes herramientas:

- 🤖 Selenium IDE  
- 📡 Postman  
- 📋 Jira / TestRail  
- ♿ WAVE Evaluation Tool  
- 🧰 Herramientas de desarrollador del navegador  
- 🎨 Validador de contraste de color  

---

# 🧩 Casos de Prueba Ejecutados

Se definieron y ejecutaron los siguientes casos de prueba:

| ID | Caso de Prueba |
|---|---|
| CP-01 | 🔎 Búsqueda de productos |
| CP-02 | 🛒 Adición de productos al carrito |
| CP-03 | 👤 Registro de usuario |
| CP-04 | ⌨️ Navegación mediante teclado |
| CP-05 | 🖼️ Verificación de texto alternativo en imágenes |
| CP-06 | 🏷️ Verificación de etiquetas en formularios |
| CP-07 | ⚠️ Validación de mensajes de error en formularios |
| CP-08 | 🎨 Verificación de contraste entre texto y fondo |
| CP-09 | 🔊 Compatibilidad con lectores de pantalla |

---

# 📊 Resultados de la Ejecución


✅ Todos los casos de prueba fueron ejecutados correctamente.  
✅ Todos los resultados obtenidos fueron **PASS**.  

No se identificaron defectos críticos que afecten la accesibilidad de las funcionalidades evaluadas.

# 🧾 Conclusión

La ejecución del plan de pruebas permitió validar que las funcionalidades evaluadas de la plataforma cumplen con criterios básicos de accesibilidad. Los usuarios pueden realizar acciones como buscar productos, agregar artículos al carrito y registrarse en la plataforma sin encontrar barreras significativas en la interacción.

Además, los resultados evidencian que la plataforma se alinea con las recomendaciones de accesibilidad definidas por **WCAG 2.1** y con la normativa colombiana vigente, particularmente la **Ley 1618 de 2013** y el **Decreto 1078 de 2015**, contribuyendo a mejorar la inclusión digital y la usabilidad del sistema.
